
MVCC 레벨

read committed -> read 하는 시간 기준으로 마지막 커밋을 읽음
repeatable read -> tx 시작 시점에서 마지막 커밋 데이터를 읽음

read uncommitted -> MVCC 는 committed 된 데이터를 읽기 떄문에 이 레벨에서는 보통 MVCC 가 적용되지 않음

LOST UPDATE 발생

repeatable read 로 바뀌게 되면 같은 데이터에 먼저 UPDATE 한 tx가 commit 되면 나중 tx는 rollback 된다
-> first-updater-win

한 tx의 레벨만 챙겨주는 것이 아닌 다른 tx의 레벨도 챙겨주어야 기대하는 rollback 이 동작함

serializable 레벨에서는 repeatable read 와 동일하게 동작
