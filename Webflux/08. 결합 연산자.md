여러 Publisher의 결과를 조합해 하나의 스트림으로 만들때 사용

# concat / concatWith
여러 Publisher를 순서대로 이어붙임
앞 Publisher 가 완료된 후 다음 Publisher 구독
순서 완벽 보장되며 동기적 방식

```
// 정적 메서드
Flux.concat(first, second, third).subscribe(...); // A B C D E F G

// 인스턴스 메서드 - 체이닝으로 이어붙이기
Flux.just("A", "B")
  .concatWith(Flux.just("C", "D"))
  .concatWith(Flux.just("E"))
  .subscribe(...);
```

# merge / mergeWith
여러 Publisher 를 동시에 구독하고 데이터가 도착하는 순서대로 합침
순서 보장 안됨
서로 다른 주기에 실시간으로 데이터를 발행하는 여러 소스를 하나로 통합할때 사용

```
// slow(500ms), fast(200ms) 동시 구독
Flux.merge(slow, fast).subscribe(...);
// fast가 먼저 완료됐으므로 fast → slow 순으로 출력

// 인스턴스 메서드
sourceA.mergeWith(sourceB).subscribe(...);
```

# zip / zipWith
각 Publisher 에서 같은 순번의 데이터를 짝지어 `Tuple` 로 조합
모든 Publisher 에서 데이터가 하나씩 도착해야 조합됨
Publisher 길이가 다른 경우 가장 짧은 Publisher 기준으로 종료되며 나머지는 버려짐
최대 8개의 Publisher 조합 가능

```
// 기본: Tuple2 반환
Flux.zip(names, ages).subscribe(...); // [홍길동, 20] [김영희, 30]

// combinator 함수로 원하는 형태 직접 조합
Flux.zip(names, ages, (name, age) -> name + " : " + age).subscribe(...);

// 3개 이상: Tuple로 받아 getT1(), getT2(), getT3() 추출
Flux.zip(names, ages, cities)
  .map(t -> t.getT1() + "(" + t.getT2() + "세, " + t.getT3() + ")")
  .subscribe(...);

// 인스턴스 메서드
names.zipWith(ages).subscribe(...);
```

# 연산자 선택 기준
|           | concat                       | merge                       | zip                     |
| --------- | ---------------------------- | --------------------------- | ----------------------- |
| 실행 방식 | 순차                         | 동시                        | 동시                    |
| 순서 보장 | O                            | X                           | O (입력 순번 기준)      |
| 조합 방식 | 단순 이어붙이기              | 도착 순서대로 합침          | 순번끼리 짝지음         |
| 사용 상황 | 페이지네이션, 순서 의존 작업 | 여러 이벤트 소스 통합, 통계 | 사용자 + 주문 정보 조합 |
