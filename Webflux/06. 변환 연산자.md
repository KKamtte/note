# map
Flux/Mono 의 각 데이터를 동기적으로 변환
새로운 스트림을 반환하므로 체이닝 가능

### 주의점
map 내부에서 블로킹 작업 (DB조회, 파일I/O 등) 수행 시 이벤트 루프 스레드가 블로킹되어 전체 성능 저하 발생
타입변환, 필드 추출 같은 경략 작업에만 사용

### 예외 처리
map 내부 예외는 Reactor 가 자동으로 onError 신호로 변환
try-catch 불필요하나 에러 핸들러는 작성해야함
예외 발생 시 이후 데이터는 처리되지 않고 스트림 종료

```
// 1, 2 처리 후 abc 에서 NumberFormatException 발생 -> 3은 처리되지 않음
Flux.just("1", "2", "abc", "3")
	.map(Integer::parseInt)
	.subscribe(
		data -> System.out.println("성공: " + data)
		error -> System.out.println("실패: " + error.getMessage())
	);
```

# flatMap
각 데이터를 Mono/Flux 로 변환하고 결과를 하나의 Flux 로 합침
내부 Publisher를 동시에 구독하므로 순서가 보장되지 않음

### delayElement 위치에 따른 동작 차이

```
// flatMap 외부 → 동기 실행 (순차 처리, 총 3초)
Flux.just("A", "B", "C")
  .flatMap(letter -> Mono.just(letter))
  .delayElements(Duration.ofSeconds(1))  // 외부
  .subscribe(...);

// flatMap 내부 → 비동기 병렬 실행 (동시 처리, 총 1초)
Flux.just("A", "B", "C")
  .flatMap(letter -> Mono.just(letter)
	  .delayElement(Duration.ofSeconds(1)))  // 내부
  .subscribe(...);
```

### concurrency 파라미터
두번째 인자로 동시 실행 개수 제한 가능
기본값 256
외부 API 나 DB 호출 시 상태 서버 부하를 고려해 적절히 설정 필요
MSA 환경에서 특히 중요하다

```
// 2개씩 묶어서 처리 → 총 3초 소요
Flux.range(1, 6)
  .flatMap(n -> Mono.just(n)
	  .delayElement(Duration.ofSeconds(1)), 2)  // 동시 2개씩 처리
  .subscribe();
```

# concatMap
flatMap과 동일하지만 내부 Publisher를 순차적으로 구독하여 입력 순서를 보장
이전 Publisher 가 완료된 후 다음 Publisher 구독

# flatMapSequential
모든 Publisher를 동시에 구독하되, 결과를 큐에 보관했다가 원래 순서대로 출력
flatMap 과 concatMap의 절충안
순서를 맞추기 위해 먼저 완료된 결과를 버퍼에 보관하므로 flatMap 보다 메모리 사용량 증가

```
Flux.just("느린", "빠른", "중간")
  .flatMapSequential(task -> Mono.just(task)
	  .delayElement(Duration.ofMillis(delay)))
  .subscribe(...);
// 실행: 동시 시작 (빠른이 먼저 완료되어도 버퍼에 대기)
// 출력: 느린 → 빠른 → 중간 (입력 순서 유지)
// 소요시간: 가장 느린 작업 기준 (concatMap보다 빠름)
```

# flatMapMany
Mono 전용 연산자
Mono 데이터를 받아 Flux 를 반환하는 1:N 변환
Mono 에서 flatMap 은 Mono만 반환 가능
map 으로 Flux를 반환하면 Mono<Flux\<T\>> 중첩 발생 -> Flux 반환이 필요하면 flatMapMany 사용

```
// 사용자 ID 하나로 여러 주문 조회 (1:N)
Mono.just("user123")
  .flatMapMany(userId -> findOrdersByUserId(userId))
  .subscribe(System.out::println);

```

# 연산자 선택 기준
| 상황                             | Mono        | Flux              |
| -------------------------------- | ----------- | ----------------- |
| 일반값 반환                      | map         | map               |
| Mono/Flux 반환 -> 순서 무관      | flatMap     | flatMap           |
| Mono/Flux 반환 -> 순서 보장      |             | concatMap         |
| Mono/Flux 반환 -> 성능/순서 절충 |             | flatMapSequential |
| Flux 반환                        | flatMapMany | flatMap 계열      |

# flatMap / concatMap / flatMapSequential 비교
|           | flatMap | concatMap | flatMapSequential |
| --------- | ------- | --------- | ----------------- |
| 내부 실행 | 병렬    | 직렬      | 병렬              |
| 순서 보장 | X       | O         | O                 |
| 성능      | 빠름    | 느림      | 중간              |
| 메모리    | 낮음    | 낮음      | 높음 (버퍼)                  |
