# Just
Mono.just() 는 즉시 실행된다 (eager)
`Mono<String> mono = Mono.just(getValue());`
이 줄이 실행되는 순간 getValue() 가 호출된다.
즉, Mono 가 만들어지기도 전에 값이 이미 계산된다.
구독을 하지 않는 상황에서도 값이 실행되는 낭비가 발생할 수 있다.
이미 계산된 값을 Mono 로 감쌀때 사용한다.

# FromSupplier
Mono.fromSupplier 는 지연 실행된다 (Lazy Evaluation)
`Mono<String> mono = Mono.fromSupplier(() -> getValue());`
구독하기 전까지 getValue() 가 호출되지 않는다.
계산을 구독 시점까지 미루고 싶을때 사용한다.

```
Mono<User> mono = Mono.fromSupplier(() -> userRepository.findById(id)); // 실행 대기
mono.subscribe(); // 이 시점에 findById() 실행
```

### FromSupplier 가 사용되는 상황
블로킹 코드를 리액티브 파이프라인에 편입시킬때 사용한다.
```
Mono.fromSupplier(() -> blockingRepository.findById(id))
      .subscribeOn(Schedulers.boundedElastic()); // 별도 스레드에서 실행
```
subscribeOn과 함께 써야 이벤트 루프 Thread를 블로킹하지 않는다. 
단독으로 사용하면 블로킹 코드가 그대로 이벤트 루프에서 실행되므로 WebFlux의 주의사항을 위반하게 된다.

### 비교 코드
```
public static void main(String[] args) {  
    Mono<String> getValue = Mono.just(getValue());  
  
    System.out.println("Mono created, but value not yet retrieved");  
  
    // fromSupplier: 구독이 발생하는 시점에 Supplier 를 실행 (Lazy Loading 과 비슷)  
    Mono<String> supplierMono = Mono.fromSupplier(() -> getValue());  
  
    System.out.println("Mono from supplier created, but value not yet retrieved");  
  
    // 이 시점에 getValue() 실행됨  
    supplierMono.subscribe(value -> System.out.println("Received: " + value));  
}  
  
static String getValue(){  
    System.out.println("Getting value...");  
    return "Hello World";  
}
```

```
Getting value... -> just 가 실행되어 getValue 가 호출된다.
Mono created, but value not yet retrieved -> fromSupplier 로 Mono 를 생성한다. getValue 는 아직 호출되지 않는다.
Mono from supplier created, but value not yet retrieved -> Mono supplier 가 생성된 로그이다.
Getting value... -> 구독이 발생하며 getValue 가 호출된다.
Received: Hello World -> 구독된 메시지를 출력한다.
```

# FromCallable
fromSupplier 와 동일하게 지연실행 되지만, Callable 인터페이스를 받기 때문에 Checked Exception 을 던질 수 있다.
예외가 발생하면 Reactor 가 자동으로 onError 신호로 변환하므로 try-catch 를 작성할 필요가 없다.
파일읽기, 외부 시스템 초기화 호출 처럼 Checked Exception 이 발생할 수 있는 작업을 Mono로 감쌀 때 사용한다.
```
public static void main(String[] args) {  
    // fromCallable: checked exception 을 던질 수 있음  
    Mono<String> callableMono = Mono.fromCallable(() -> {  
        if (Math.random() > 0.5) {  
            throw new Exception("Random exception occurred");  
        }  
  
        return "success";  
    });  
  
    callableMono.subscribe(  
            result -> System.out.println("결과: " + result),  
            error -> System.err.println("에러: " + error.getMessage())  
    );  
}
```

# Defer
fromSupplier 는 값의 생성을 지연하는데, defer 는 Mono 자체의 생성을 지연한다.
구독할 때 마다 람다가 새로 실행되어 새로운 Mono 인스턴스가 만들어진다.

### 코드1
```
public static void main(String[] args) throws InterruptedException {  
    Mono<Long> deferedMono = Mono.defer(() -> {  
        System.out.println("created...");  
        return Mono.just(System.currentTimeMillis());  
    });  
  
    System.out.println("Mono created");  
    deferedMono.subscribe(time -> System.out.println("first subscribe: " + time));  
  
    Thread.sleep(1000);  
  
    System.out.println("1 sec later");  
    deferedMono.subscribe(time -> System.out.println("second subscribe: " + time));
}
```
결과
```
Mono created -> 이 시점엔 아무것도 실행되지 않는다
created... -> 첫 구독시 람다가 실행된다
first subscribe: 1780995690000
1 sec later
created... -> 두번째 구독시 람다가 다시 실행된다
second subscribe: 1780995691009 -> 첫 구독의 결과 값과 다른 값이 나온다
```

### 코드2
```
static boolean featureEnabled = false;

public static void main(String[] args) {  
	Mono<String> conditionMono = Mono.defer(() -> {  
	    if (featureEnabled) {  
	        return Mono.just("Feature enabled");  
	    } else {  
	        return Mono.empty();  
	    }  
	});  
	  
	System.out.println("Feature enabled: " + featureEnabled);  
	conditionMono.subscribe(  
	        result -> System.out.println("Result: " + result),  
	        error -> System.out.println("Error: " + error),  
	        () -> System.out.println("Completed without emitting any value")  
	);  
	  
	featureEnabled = true;  
	conditionMono.subscribe(  
	        result -> System.out.println("Result: " + result),  
	        error -> System.out.println("Error: " + error),  
	        () -> System.out.println("Completed without emitting any value")  
	);
}
```
결과
```
Feature enabled: false
Completed without emitting any value
Result: Feature enabled
Completed without emitting any value
```
comditionMono 변수 값이 구독 시점의 featuredEnabled 상태에 따라 결과 값이 다르게 나타난다.

### fromSupplier 와의 차이
|                            | fromSupplier | defer     |
| -------------------------- | ------------ | --------- |
| 반환 타입                  | T            | Mono\<T\>   |
| 지연 대상                  | 값           | Mono 자체 |
| 조건에 따라 다른 Mono 반환 | 불가         | 가능      |

### defer 가 필요한 이유
상황에 따라 Mono.just, Mono.empty, Mono.error 중 다른 걸 반환해야 할 때 필요하다.
```
// fromSupplier 로는 불가능
Mono.defer(() -> {
	if (featuredEnabled) {
		return Mono.just("data");
	} else {
		return Mono.empty();
	}
});
```

### 토큰 갱신 사용 패턴
```
Mono<String> tokenMono = Mono.defer(() -> {
	if (token.isExpired()) {
		return refreshToken(); // 만료 시 갱신 Mono 반환
	}
	return Mono.just(token.getValue()); // 유효 시 캐시된 값 반환
});
```

# Chaining
### Map - 동기 변환 T -> R
Mono 가 발행하는 데이터를 동기적으로 변환한다.
원래 Mono를 변경하는 것이 아니라 새로운 Mono를 반환하므로, map을 여러번 체이닝할 수 있다.

```
Mono<String> originMono = Mono.just("hello");

// 새로운 Mono를 반환 -> originMono는 여전히 "hello"를 발행
Mono<String>upperMono = originMono.map(String::toUpperCase);
upperMono.subscribe(s -> System.out.println("Uppercase: " + s)); // HELLO

// 체이닝: map을 연속으로 연결
Mono<Integer> length = Mono.just("hello reactor")
		.map(s -> s.toUpperCase()) // T(String) -> R(String)
		.map(s -> s.length());     // T(String) -> R(Integer)
length.subscribe(s -> System.out.println("Length: " + s)); //13
```

### FlatMap - 비동기 변환 T -> Mono\<R\>
반환 함수가 Mono를 반환할 때 사용한다.
map을 사용하면 `Mono<Mono<R>` 이중 중첩이 생기는데, flatMap 은 이를 평탄화하여 `Mono<R>` 로 변환한다.

```
Mono<String> result = Mono.just("1")
		.flatMap(userId -> findUserName(userId)); // T -> Mono<R>
result.subscribe(data -> System.out.println("result: " + data)); // jackson

// map을사용했을 때의 문제점
Mono<Mono<String>> wrong = Mono.just("2")
		.map(userId -> findUserName(userId)); // Mono<Mono<String>> 이중 중첩 발생
		
static Mono<String> findUserName(String userId) {
	if (userId.equals("1")) {
		return Mono.just("jackson");
	} else {
		return Mono.just("samuel");
	}
}
```

### Map vs FlatMap 선택 기준
| 상황                                           | 사용 연산자 |
| ---------------------------------------------- | ----------- |
| 변환 함수가 일반 값(T)을 반환                  | `map`       |
| 변환 함수가 Mono를 반환 (DB 조회, API 호출 등) | `flatMap`   |
