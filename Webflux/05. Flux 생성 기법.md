0개 이상(0~N)의 데이터를 비동기적으로 발행하는 Reactive 타입
데이터 발행 순서가 보장됨 (시퀀셜 처리 가능)
사용상황: DB 목록 조회, 실시간 이벤트 스트림(채팅/센서/주가), 파일 줄 단위 읽기

# fromStream
Java Stream 으로 부터 Flux 생성
Java Stream 은 한번만 소비 가능하므로, 이 Flux도 한번만 구독 가능
두번 구독시 IllegalStateException 발생
여러번 구독이 필요하면 Supplier 람다로 감싸서 구독마다 새 Stream 생성

```
// 단건 소비만 가능
Flux<Integer> flux = Flux.fromStream(Stream.of(10, 20, 30));

// 다중 구독 가능
Flux<Integer> flux = Flux.fromStream(() -> Stream.of(10, 20, 30));
```

# generate
상태를 유지하면서 데이터를 하나씩 동기식으로 생성
3가지 요소로 구성:
- 1번째 인자: 초기 상태를 제공하는 Supplier
- 2번째 인자: (현재 상태, SynchronousSink) 를 받는 BiFunction
	- sink.next(): 데이터 발행
	- sink.complete(): 완료 신호
	- sink.error(): 에러 신호
	- 반환값이 다음 호출의 상태가됨
제약: 한번 호출에 sink.next() 는 최대 1번만 호출 가능. 2번 이상 호출 시 예외 발생
```
Flux.generate(
	() -> 0,
	(state, sink) -> {
		sink.next(state);
		if (state == 4) {
			sink.complete();
		}
		return state + 1;
	}
)
```

# create
generate 보다 더 유연한 방식. FluxSink를 통해 자유롭게 데이터 발행
- 한번에 sink.next() 여러번 호출 가능
- 비동기 작업과 결합 가능
- 콜백 기반 API 를 Reactive Streams 로 변환할때 유용

```
Flux.create(sink -> {
	sink.next("first");
	sink.next("second);
	sink.complete();
})
```

### generate 와 비교
|                | generate  | create                |
| -------------- | --------- | --------------------- |
| next 호출 횟수 | 1회       | 제한 없음             |
| 방식           | 동기      | 동기/비동기 모두 가능 |
| 상태 관리      | 내장 지원 | 직접 관리                      |

# 변환 / 필터링
### map
Mono의 map과 동일
Flux의 각 데이터에 동기적으로 변환 적용
체이닝 가능

```
Flux.just("hello", "world", "reactor)
	.map(String::toUpperCase)
	.map(String::length)
```

### flatMap
각 데이터에 대해 Mono나 Flux를 반환하고, 결과를 하나의 Flux로 합침
1:N 변환도 가능

```
// 1:1 (DB 조회)
Flux.just("1", "2", "3")
	.flatMap(userId -> findUserName(userId));
	
// 1:N (문자열 -> 각 문자)
Flux.just("apple", "banana")
	.flatMap(word -> Flux.fromArray(word.split("")));
```
flatMap 은 내부 Publisher의 결과 순서를 보장하지 않음
비동기 환경에서 순서가 섞일 수 있음

### filter
조건이 true인 데이터만 다음 단계로 전달
false인 데이터는 제거
Java Stream 의 filter 와 동일한 개념

```
// 짝수만
Flux.range(1, 10).filter(i -> i % 2 == 0);

// a로 시작하는 단어
Flux.just("apple", "banana", "avocado")
	.filter(s -> s.startsWith("a"))
	.map(String::toUpperCase);
```

그외에도 take(), skip() 등도 존재하나 자주 사용되지는 않는다

# Flux -> Mono 전환
Flux 와 Mono는 동일한 표준 인터페이르 구현체이므로 상호 전환 가능
| 메서드        | 방향                  | 설명                                                                            |
| ------------- | --------------------- | ------------------------------------------------------------------------------- |
| collectList() | Flux -> Mono<List<T>> | 모든 데이터를 List로 수집. 완료까지 메모리에 보관하므로 데이터 많을 시 OOM 주의 |
| next()        | Flux -> Mono<T>       | 첫번째 데이터만 Mono로 가져옴. 수신 후 나머지 취소                              |
| flux()        | Mono -> Flux          | Mono를 Flux로 변환. 데이터 없으면 빈 Flux                                       |
| count()       | Flux -> Mono<Long>    | 발행된 데이터 총 개수 반환                                                                                |
