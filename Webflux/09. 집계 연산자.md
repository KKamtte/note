Flux의 데이터를 소비해서 최종적인 `Mono` 하나를 반환
집계이므로 결과는 항상 `Mono` 가 나온다

# Reduce
모든 데이터를 누적 함수로 순서대로 결합해 하나의 값으로 축소
첫번째 데이터를 초기 누적값으로 사용하고 두번째 데이터부터 누적 함수 적용한다

```
// 1 → 1+2=3 → 3+3=6 → 6+4=10 → 10+5=15
Flux.just(1, 2, 3, 4, 5)
  .reduce((acc, next) -> acc + next)
  .subscribe(...); // 15

// HelloReactorWorld
Flux.just("Hello", "Reactor", "World")
  .reduce((acc, next) -> acc + next)
  .subscribe(...);
```

### 초기값 지정
두번째 인자로 초기값이 설정 가능하다
빈 Flux 인 경우 초기값이 결과로 반환되며 초기값 없는 reduce는 빈 Flux에서 `Mono.empty()` 를 반환한다

```
// 115
Flux.just(1, 2, 3, 4, 5).reduce(100, (acc, next) -> acc + next)
  .subscribe(...);

// 100
Flux.<Integer>empty().reduce(100, (acc, next) -> acc + next)
  .subscribe(...);
```

### 빈도수 집계 활용 예시

```
// {apple=2, banana=2, cherry=1}
Flux.just("apple", "banana", "apple", "cherry", "banana")
  .reduce(new HashMap<String, Integer>(), (map, word) -> {
	  map.merge(word, 1, Integer::sum);
	  return map;
  })
  .subscribe(...);
```

# Scan
reduce와 동일한 누적 로직이지만, 최종 결과만 반환하는 reduce와 달리 중간 누적 결과도 모두 발행한다
반환 타임은 Flux 이다

```
// 15
Flux.just(1, 2, 3, 4, 5).reduce((acc, next) -> acc + next)
  .subscribe(...); 
// 1, 3, 6, 10, 15
Flux.just(1, 2, 3, 4, 5).scan((acc, next) -> acc + next)
  .subscribe(...); 
```

|           | reduce    | scan                        |
| --------- | --------- | --------------------------- |
| 반환 타입 | Mono      | Flux                        |
| 발행 결과 | 최종값    | 중간값 모두                 |
| 사용 상황 | 최종 집계 | 실시간 누적 통계, 과정 확인 |

# groupBy
키 추출 함수 기준으로 데이터를 실시간으로 그룹화
모든 데이터를 기다리지 않고 도착하는 즉시 그룹으로 분류
반환 타입은 `Flux<GroupedFlux<K, V>>`, `GroupedFlux` 는 `key()` 로 그룹 키 제공

```
// 홀수: [1, 3, 5, 7, 9]
// 짝수: [2, 4, 6, 8, 10]
Flux.range(1, 10)
  .groupBy(n -> n % 2 == 0 ? "짝수" : "홀수")
  .flatMap(group -> group.collectList()
	  .map(list -> group.key() + ": " + list))
  .subscribe(...);
```

`GroupedFlux` 는 반드시 소비되어야 한다
소비하지 않으면 내부 버퍼에 데이터가 쌓여 메모리 문제가 발생한다
`flatMap` 을 통해 내부에서 즉시 처리하는 패턴이 좋다
