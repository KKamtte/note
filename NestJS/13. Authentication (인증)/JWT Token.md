# JWT Token 이란
- 유저의 정보를 Base 64로 인코딩된 String 값에 저장하는 도구

## JWT Token 의 특징
- JWT Token 은 Header, Payload, Signature 로 구성되어 있으며 Base 64 로 인코딩 되어 있다.
- JWT Token 은 서버에서 생성되며 클라이언트에 저장된다.
	- 서버에서 정보를 담아서 JWT Token 을 생성한다음 클라이언트로 전달
	- 클라이언트는 토큰을 가지고 다음 요청을 보내게 됨
- 클라이언트에서 요청을 보낼 때 JWT Token ID 를 같이 보내면 현재 요청을 보내는 사용자가 누구인지 서버에서 알 수 있다.
	- 요청마다 매번 사용자의 ID / PW 를 물어볼 필요가 없다.
	- 발급받은 토큰을 통해 해당 사용자를 유추할 수 있다.
- JWT Token 은 데이터베이스에 저장되지 않고 Signature 값을 이용해서 검증할 수 있다.
	- Header와 Payload 가 변형되지 않았다는 것을 Signature 를 통해 검증
	- 검증할 때 마다 매번 데이터베이스를 접근할 필요가 없다.
- 정보가 모두 토큰에 담겨있고 클라이언트에서 토큰을 저장하기 때문에 정보 유출의 위험이 있다.
	- Token 에 사용자의 정보가 담겨져 있기 때문에 Token이 탈취당하면 보안 위험이 있다.
	- Token 에 저장되는 데이터가 유출이 되도 문제가 없는 정보면 크게 영향은 없다.
	- Refresh Token 및 Access Token 을 이용해 사용자 확인도 가능하다.
- 데이터베이스가 필요없기 때문에 Horizontal Scaling 이 쉽다.
